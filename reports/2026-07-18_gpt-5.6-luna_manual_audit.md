## Major benchmark issues

- `airport_t04`, `airport_t05`, `airport_t13`: the shared travel-service lookup performs hidden exact string matching on `service_type`; natural equivalents fail against serialized values such as `lost_property` or `assistance`.
- `telecom_t14`: the decisive billing-instruction tool crashes because of an undefined helper, making the payment-resolution path non-executable.

## Cross-domain findings

1. **Monolithic final-answer regexes are the dominant evaluator failure mode.** Correct actions and decisions are frequently rejected because one narrow wording, an unnecessary clause, or a harmless contextual mention does not match.
2. **Non-decision-critical page requirements create false negatives.** Exact secondary pages should not be mandatory when the governing evidence and decisive state are already established.
3. **Hidden tool serialization is a benchmark defect.** Tool schemas should expose enums or normalize spaces, underscores, hyphens, case, and documented synonyms.
4. **Plain-text questions are not executable interaction.** When continuation depends on the answer, the agent must call `get_user_help` rather than merely asking in its final text.
5. **Weakened certainty can change the decision.** “Likely” or “probably” is a real failure when the available state conclusively establishes eligibility, liability, or a fee trigger.
6. **Timing, authorization expansion, and checkpoint refresh should remain strict.** The strongest genuine failures involved premature action, stale state, or unapproved financial/transactional consequences.

## Recommended changes

1. Replace monolithic final-answer regexes with independent semantic slots plus explicit forbidden claims.
2. Require only decision-critical sources and accept equivalent official URLs and delegated endpoints.
3. Normalize tool arguments centrally and publish valid enum values.
4. Add task-tool smoke tests and fail benchmark builds on runtime exceptions.
5. Add all 25 confirmed false-negative trajectories as evaluator regression fixtures.
6. Preserve strict action-success, authorization, duplicate-submission, and post-checkpoint refresh checks.

## Final assessment

Automatic utility substantially underestimates performance in this run: manual success increases from 35 to 60 after correcting 25 false negatives. Evaluator robustness is the highest-priority fix, followed by the four major tool/benchmark defects. Genuine agent failures remain concentrated in missing authoritative grounding, premature user escalation, failure to use `get_user_help`, weak binding to decisive state, and unsafe or premature action timing.
